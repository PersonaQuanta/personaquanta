Overview

How PersonaQuanta works

Four capabilities work together to turn a short questionnaire into a meaningful, safe, and explainable financial self-model.

◎

Pseudonym-first

  • No real name required to start
  • Inputs are entirely user-controlled
  • Reduces friction at step 1
  • No bank logins or document uploads
◈

Explainable Top Drivers

  • Plain-language: why the result moved
  • Constraints highlighted (what limits upside)
  • Reduces "black box" distrust
  • 3 drivers + 1 constraint per report
◐

CONF + uncertainty ranges

  • Range-first outputs — no false precision
  • CONF tied to input completeness
  • Missing data widens ranges transparently
  • Users understand what they don't know
⟳

What-if exploration

  • One variable change → directional impact
  • Users learn trade-offs safely
  • "Finish to see the result" engagement loop
  • 3 presets: income, debt, coverage

Valuation approach

How ranges are derived (no magic)

No external price feeds. No bank logins. Ranges are built from user inputs and conservative assumptions — always CONF-anchored.

User-entered values

If users provide asset values directly, those inputs are used as-is with a manual confirmation step. No auto-estimation without user acknowledgment.

Conservative defaults

When fields are missing or unknown, the model defaults to conservative range assumptions — not market averages. CONF label widens accordingly.

Liquidity haircuts

Illiquid asset categories (e.g. real estate, private holdings) apply conservative liquidity haircuts to avoid over-stating net worth ranges.

CONF widening rules

Each missing or inconsistent field increases CONF uncertainty. Users see exactly why their range is wide — reducing black-box perception.

Manual override

Users can override any auto-estimate with their own figure. All overrides are flagged in the report as user-confirmed values.

No market accuracy claims

Outputs are engagement tools — not valuations. We never claim market accuracy. CONF ranges communicate the limits of the model explicitly.
